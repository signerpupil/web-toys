Wichtigste Erkenntnisse aus der Codebase-Analyse

✓
KI-Infrastruktur existiert bereits — Infrastructure/AI/Prompt/ mit GPT_OSS_120B fuer Foerderziel-Generierung. PromptBuilder-System mit Temperature, MaxTokens, JSON Schema Response. Direkt erweiterbar fuer neue Features.
✓
Duplikaterkennung implementiert — PersonDuplicate Modul mit Rules, Comparators, Merge-Operationen. Feature #33 benoetigt nur Verbesserung, nicht Neuimplementierung.
✓
Azure Translator konfiguriert — AZURE_TRANSLATION_URL und AZURE_TRANSLATION_REGION sind als Environment-Variablen vorhanden. Feature #20 (Schuluebersetzung) kann darauf aufbauen.
✓
Sitzungs-Modul mit reichen Textfeldern — Meeting/Protocol mit AgendaItem (content, fact, consideration, proposal, resolution) — idealer Kandidat fuer KI-gestuetzte Protokollgenerierung.
⚠
Kein Chat-System im Backend — PUPIL Connect scheint ein separates System zu sein. Im Backend gibt es nur NotificationCenter + SendGrid Mail. Features #6 und #15 (Chat-basiert) sind deutlich aufwaendiger als angenommen.
⚠
Keine Volltextsuche — Kein Elasticsearch/Meilisearch. Nur MySQL-basierte Queries. RAG-Features (#7, #22, #30) benoetigen externe Infrastruktur.
ℹ
Keine eigenstaendige LP21-Datenstruktur — LP21-Kompetenzen sind nur als has_competence_area Flag abgebildet. Fuer LP21-Lektionsvorschlaege muss LP21-Daten extern angebunden werden (RAG).

Wichtige Einschraenkungen fuer KI-Features

Kein Chat-System

PUPIL Connect ist ein separates System. Im Backend nur NotificationCenter + SendGrid. Chat-basierte KI-Features benoetigen separate Integration.

Keine Volltextsuche

Kein Elasticsearch/Meilisearch. RAG-Features benoetigen externe Vector-DB. Fuer einfache Suche genuegt MySQL LIKE.

DB-basierte Queue

ProcessQueue in MySQL. Fuer KI-Features mit hohem Throughput Redis aktivieren (bereits in Config vorhanden).

Kein LP21-Datenmodell

LP21-Kompetenzen nur als has_competence_area Flag. LP21-basierte Features benoetigen externe Datenquelle (RAG ueber LP21-Dokumente).
