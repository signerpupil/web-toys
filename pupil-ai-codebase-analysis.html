Implementierungsstrategie

Woche 1-2: AI Gateway

Bestehende Infrastructure/AI/Prompt/ als Basis. Redis-Cache aktivieren (bereits konfiguriert). Rate Limiting, Token-Tracking, Audit-Log. Model-Router fuer verschiedene LLM-Provider.

Woche 3-4: ProcessQueue fuer KI

Bestehende ProcessQueue nutzen fuer langlaufende KI-Aufgaben. Neuer AiQueueHandler Typ. Ergebnis-Caching in S3/NFS.

Woche 3-6: Vue AI-Components

Wiederverwendbare AI-Komponenten in libs/ai/: AiSuggestionButton, AiTextCompletion, AiSummaryPanel. Pinia Store fuer AI State.

Fortlaufend: FunctionFrom erweitern

Neue KI-Funktionen als FunctionFrom Enum-Werte registrieren. Jedes Feature = neuer Enum-Wert + PromptPreparationData + System-Prompt.

Key Findings & Ueberraschungen

Wichtigste Erkenntnisse aus der Codebase-Analyse

✓
Symfony 6.4, nicht Laravel — Das Framework ist Symfony mit DDD-Pattern, nicht Laravel wie urspruenglich angenommen. Dies aendert die Integrationsstrategie (Event Dispatcher statt Laravel Events, Phinx statt Laravel Migrations).
✓
KI-Infrastruktur existiert bereits — Infrastructure/AI/Prompt/ mit GPT_OSS_120B fuer Foerderziel-Generierung. PromptBuilder-System mit Temperature, MaxTokens, JSON Schema Response. Direkt erweiterbar fuer neue Features.
✓
Duplikaterkennung implementiert — PersonDuplicate Modul mit Rules, Comparators, Merge-Operationen. Feature #33 benoetigt nur Verbesserung, nicht Neuimplementierung.
✓
Azure Translator konfiguriert — AZURE_TRANSLATION_URL und AZURE_TRANSLATION_REGION sind als Environment-Variablen vorhanden. Feature #20 (Schuluebersetzung) kann darauf aufbauen.
⚠
Kein Chat-System im Backend — PUPIL Connect scheint ein separates System zu sein. Im Backend gibt es nur NotificationCenter + SendGrid Mail. Features #6 und #15 (Chat-basiert) sind deutlich aufwaendiger als angenommen.
⚠
Keine Volltextsuche — Kein Elasticsearch/Meilisearch. Nur MySQL-basierte Queries. RAG-Features (#7, #22, #30) benoetigen externe Infrastruktur.
⚠
DB-basierte Queue — Custom ProcessQueue in MySQL, kein Redis/RabbitMQ. Fuer KI-Features mit hohem Throughput sollte Redis aktiviert werden (bereits in Config vorhanden).
ℹ
Keine eigenstaendige LP21-Datenstruktur — LP21-Kompetenzen sind nur als has_competence_area Flag auf Subject und CurriculumStatus abgebildet. Fuer Features wie #18 (LP21-Lektionsvorschlaege) muss LP21-Daten extern angebunden werden (RAG).
